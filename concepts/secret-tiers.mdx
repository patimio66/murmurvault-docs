---
title: "Secret tiers"
description: "The difference between soft and top secrets, and when to use each."
---

Every secret belongs to one of two tiers. The tier decides how the value is encrypted and what it takes to reveal it.

## Soft secrets

Soft is the default tier. Soft values are encrypted with the [Master Key](/concepts/security-model#keys).

- Revealed whenever the vault is unlocked — by passphrase or any linked passkey.
- Decrypted in place when you open the secret.
- Suitable for the majority of credentials.

## Top secrets

Top values are encrypted with the separate [Top Key](/concepts/security-model#keys), which only passkeys hold. Unlocking the vault is not enough to read them.

Revealing a top secret requires, every time:

1. A **one-time code** emailed to your address.
2. A **passkey tap** to release the Top Key.

This adds a second factor (your email) and a hardware-backed factor (your passkey) on top of an already-unlocked vault.

<Note>
  The top tier is only available once a passkey holds the Top Key. Until then, the top option is disabled when creating a secret. Enable it in [Settings → Vault](/settings/vault).
</Note>

## Choosing a tier

<CardGroup cols={2}>
  <Card title="Use soft for" icon="unlock">
    Everyday logins, API keys you reach for often, notes — anything where unlocking the vault is protection enough.
  </Card>
  <Card title="Use top for" icon="lock">
    Root credentials, master recovery codes, production keys — secrets where you want a per-reveal email code and passkey tap.
  </Card>
</CardGroup>

## Limitations of top secrets

- **No editing.** Top values cannot be edited in place. To change one, delete it and create it again.
- **No bulk reveal.** Each top secret is revealed individually, within a short window after verification.
- **Passkey required.** If no passkey holds the Top Key, top secrets cannot be created or read.

See [Top-tier secrets](/vault/top-secrets) for the full reveal walkthrough.

## How tiers look to the server

The server knows only how many secrets are soft and how many are top — it counts blobs by tier. Names, types, and values are sealed inside the ciphertext and are never visible. See [Security model](/concepts/security-model#what-the-server-can-see).
