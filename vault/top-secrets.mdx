---
title: "Top-tier secrets"
description: "Enable the top tier and reveal the highest-protection secrets."
---

Top secrets are encrypted with the [Top Key](/concepts/security-model#keys), held only by passkeys. Unlocking the vault is not enough to read them — each reveal requires an emailed code and a passkey tap.

Use the top tier for your most sensitive values: root credentials, master recovery codes, production keys.

## Enable the top tier

The top tier becomes available once a passkey holds the Top Key.

1. Go to [Settings → Vault](/settings/vault).
2. Grant the Top Key to a linked passkey (the **top-secret passkey** action).
3. The top tier is now selectable when you create a secret.

<Note>
  Only passkeys can hold the Top Key. Your passphrase can never wrap it, so a passphrase alone can never reveal a top secret.
</Note>

## Create a top secret

In the secret form, choose the **top** tier (available only once a passkey holds the Top Key), then fill in the fields as usual. The value is encrypted with the Top Key before sending.

## Reveal a top secret

With the vault unlocked:

<Steps>
  <Step title="Request a code">
    Open the top secret and request a reveal code. A **6-digit code** is emailed to your account address.

    The code is valid for **10 minutes**, single-use, and allows up to **5 attempts** before it is burned and you must request a new one.
  </Step>
  <Step title="Enter the code">
    Type the code from your email. A correct code opens a short reveal window for this secret.
  </Step>
  <Step title="Tap your passkey">
    Authenticate with a passkey that holds the Top Key. This releases the Top Key just long enough to decrypt the value.
  </Step>
  <Step title="View the value">
    The plaintext is shown and kept only in the page's memory. It is discarded when the vault locks.
  </Step>
</Steps>

The reveal window is short (180 seconds by default) and applies per reveal. After it closes — or when the vault locks — you must start over to view the value again.

## Limitations

- **No editing.** To change a top secret, delete it and create a new one.
- **One at a time.** Each top secret is revealed individually.
- **Email required.** Reveals depend on receiving the one-time code, so you need access to your account email.

## Deleting passkeys with top secrets

The Top Key lives only in passkey wrappers. If you delete the last passkey holding it while top secrets still exist, those secrets become permanently unreadable — so murmurvault **blocks** removing the last Top-Key passkey while any top secret remains.

<Warning>
  Keep more than one passkey holding the Top Key. If you lose your only Top-Key passkey, every top secret in that vault is unrecoverable.
</Warning>
