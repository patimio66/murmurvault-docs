title: "Manage secrets"
description: "Add, edit, reveal, copy, search, and delete secrets."
---

Secrets are the items stored in a vault. Each secret has a name, a type, a [tier](/concepts/secret-tiers), and a set of fields. Everything except the tier is encrypted in your browser before it reaches the server.

Unlock the vault before working with secrets. See [Unlocking](/vault/unlocking).

## Secret types

| Type | Fields | Notes |
|------|--------|-------|
| **KV** | value, notes | A single key/value pair. The value is masked. |
| **Login** | username, password, URL, notes | The URL field offers an open-in-new-window action. |
| **Recovery Codes** | codes (one per line) | Multi-line list of backup codes. |
| **Secure Note** | note | A free-form encrypted note. |

Sensitive fields are masked by default with a reveal toggle, and several offer copy-only access so you can use a value without displaying it.

## Add a secret

1. Choose **Add secret**.
2. Pick a type and, if available, a [tier](/concepts/secret-tiers) (soft or top).
3. Enter a **name** and fill in the fields.
4. Save.

The name and all field values are encrypted client-side. A name is required.

<Note>
  The **top** tier is selectable only when a passkey holds the Top Key. Otherwise the option is disabled — enable it in [Settings → Vault](/settings/vault).
</Note>

## Reveal and copy

- Open a secret to view its details.
- Toggle a masked field to reveal its value, or use the copy action to put it on the clipboard without showing it.
- For Login secrets, the URL field can open directly in a new window.

Soft secrets reveal as soon as the vault is unlocked. Top secrets need an extra step — see [Top-tier secrets](/vault/top-secrets).

## Edit a secret

Open a **soft** secret and edit its name or fields, then save. The updated values are re-encrypted before sending.

<Warning>
  **Top secrets cannot be edited.** To change one, delete it and create it again.
</Warning>

## Search

Use the search palette to jump to a secret by name across the open vault. Because the server cannot read secret names, search runs in your browser over already-decrypted metadata while the vault is unlocked.

## Delete a secret

Open the secret and choose delete. Deletion is immediate; the encrypted blob is removed from the server. There is no trash.
